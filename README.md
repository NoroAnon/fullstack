# fullstack
Testeo de Flujos
proyecto full stack que tendrá login, validara usuarios por rol, la principal función es que sera un validador de Apis, por ejemplo cada usuario tendrá su inicio dividido por proyectos, en el cual una vez que entre a cada proyecto tendrá apartado ya sea de Apis u otros flujos entonces si entra a un api deberá ya sea agregar una nueva o mostrar el estatus del api
ya que la finalidad del proyecto es tener un testeo de la misma e igual si entra a los otros apartados
esto para ver si responde bien el flujo que se tiene pensado que hace 
en las Apis se tiene que validar por ejemplo si se le mandan strings y los campos son int ver que responde 
e igual si los campos son strings y se le manda strings pero con información incorrecta ver que responde y ver si regresa código de error y sino regresa código de error igual poner como alertas en el front de que esa api no regresa code o código 
e igual se pondrá un apartado antes de testear el api de si se quiere que se compare el código a esto las apis back del proyecto van a ir a una bd a traer el code con su descripción y compararla sino están pues mostrara que no hay code 

el proyecto debe de ser bd apis back y front 
e igual debe de tener un apartado para estresar las apis (con sus métricas, si tardo en responder y eso)
e igual otro donde se tendrá diagrama de flujo de la misma ya sea que se muestre una imagen o crear uno (este me interesa)
